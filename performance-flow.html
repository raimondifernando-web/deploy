0
Validación de datos
Antes de procesar nada, validate-meta-data.py verifica calidad, frescura y límites de los datos crudos y procesados. Genera reporte de calidad que bloquea el pipeline si hay anomalías.
script: validate-meta-data.py
Qué valida: Datos raw vs procesados, frescura (timestamps), bounds checks (valores fuera de rango), integridad de campos obligatorios
Output: quality-report.json — si hay errores críticos, el pipeline se detiene y notifica
Frecuencia: Corre como primer paso de daily-engine.sh antes de cualquier extracción o transformación

Validación contra Knowledge Bases
validate-meta-data.py valida calidad de los datos procesados contra las KBs. history-manager.py guarda snapshot diario del estado actual para comparación futura.
script: history-manager.py
validate-meta-data.py: Verifica datos contra 20 reglas de Meta Ads — valores esperados, rangos normales, consistencia temporal
history-manager.py: Guarda snapshot diario (métricas actuales + estado de campañas) como punto de comparación para el review de día +3
Fuentes KB: meta-knowledge-base.md (20 reglas) + performance-playbook.md + research-performance-metrics.md
Se actualiza: Semanalmente (scheduled task lunes 10:00 AM) + sync bidireccional con Notion
Claude Code — meta-knowledge-base.md Meta Docs — fuente oficial Memory MCP — reglas críticas
3
Insights → Learnings (PL)
generate-actions.py detecta patrones en los datos: fatiga creativa (CTR -20% + freq >3x en 14 días), tendencias rolling 90 días, anomalías. Los hallazgos se convierten en Learnings validados por KB.
script: generate-actions.py
Ejemplo: PL-11 "TOFU optimiza por Visitas Perfil — solo 25 outbound clicks de $864K invertidos"
Tipos de learning: Observación → Acción tomada → Resultado → Patrón confirmado
KB-validado: Cada learning se contrasta contra meta-knowledge-base.md antes de guardarse en Notion
Notion — Performance Learnings Claude Code
4
Acciones automáticas → Acciones del Día (AC)
generate-actions.py aplica 8 reglas de detección (fatiga, low performer, MER, frecuencia, saldo, pixel). Valida cada acción contra KB + anti-patrones, deduplica, y crea en Notion.
script: generate-actions.py
Ejemplo: AC-5 "Cambiar TOFU a Landing Page Views" → resuelve PL-1 + PL-11
Incluye: Cómo ejecutar (pasos concretos), impacto esperado, comando técnico
Anti-patrones: Si la acción fue ejecutada antes y empeoró resultados → bloqueada automáticamente
KB-validado: Checklist de 20 reglas Meta Ads antes de crear cualquier AC en Notion
Notion — Acciones del Día Reporte Web (Vercel)
5
Fernando aprueba
Revisás las acciones en Notion y decidís: aprobar, posponer o rechazar. Podés aprobar varias a la vez.
manual
Cada acción tiene dos columnas clave:
Claude puede (SI / NO)
Lo llena Claude. Indica si tiene capacidad tecnica para ejecutar via API. No lo podes cambiar.
Ejecutar con Claude (tilde)
Lo controlas vos. Si lo tildas y aprobas, Claude ejecuta automaticamente. Si no, lo haces manual.
Combinaciones posibles:
Claude puede SI + Tilde ON + Aprobada → Claude ejecuta automaticamente
Claude puede SI + Tilde OFF + Aprobada → Fernando ejecuta manual
Claude puede NO → Siempre manual (requiere accion humana: diseño, desarrollo, etc.)
Notion — Cambiar estado + tildar Claude Code — dar la orden Claude Chat — dar la orden
6
Claude ejecuta + registra
execute-actions.py cambia budgets, audiencias o creativos via Meta Ads API. log-action.py registra cada ejecución en el Execution Log de Notion (EX) con snapshot de métricas antes.
script: execute-actions.py
Por cada acción:
1. Ejecuta el cambio en Meta Ads
2. Marca AC → 🚀 Ejecutada
3. Crea EX con métricas "antes" como snapshot
4. Actualiza TODOS los PL vinculados → "Pendiente de resultado"
5. Programa revisión para 3 días después
Claude Code — execute-actions.py Meta Ads API — cambios Notion — Execution Log
7
Día +3: Revisión automática de resultados
review-results.py compara snapshot del día de ejecución vs hoy. Clasifica: ✅ Positivo, ➡️ Neutro, ❌ Negativo. Los negativos generan anti-patrones.
script: review-results.py
Actualiza EX: Métricas después + Impacto medido (Positivo/Neutro/Negativo)
Actualiza PL: Resultado observado + si se confirma → "Patrón confirmado"
Claude Code — revisión auto Meta Ads API — métricas Notion — Execution Log Notion — Learnings
8
Aprendizaje → Playbook + Thresholds
update-thresholds.py ajusta scoring automáticamente cuando 3+ learnings confirman un patrón. update-knowledge-base.py incorpora los aprendizajes confirmados al playbook local.
script: update-thresholds.py
Feedback loop cerrado:
• Confirmado → update-knowledge-base.py actualiza playbook + update-thresholds.py ajusta scoring (3+ confirmaciones)
• Empeoró → anti-patterns.json → generate-actions.py NO repite la misma acción
• Sin efecto → nota para re-evaluación en próximo ciclo
Claude Code — Playbook local Memory MCP — todas las sesiones Notion — Patrones confirmados
9
Notificación → Notion + Pipeline Summary
notify-summary.py + generate-pipeline-summary.py consolidan TODOS los artefactos del día en un resumen para Fernando: acciones tomadas, learnings nuevos, métricas clave, próximos pasos.
script: notify-summary.py
Consolida: AC ejecutadas, PL creados/actualizados, EX del día, alertas de quality-report.json, thresholds ajustados
Output: pipeline-summary.json → página Notion actualizada automáticamente → Fernando ve TODO sin pedir
generate-pipeline-summary.py: Genera vista ejecutiva diaria con estado del sistema completo
